How To Use The AI4U Routing OS

A step-by-step guide to routing every task to the perfect AI platform — every time.

STEP 01 / 07
🎯
What Is the AI4U Routing OS?
The AI4U Routing OS is an intelligent decision framework that analyzes your task and routes it to the single best AI platform for that specific job. No more guessing, no more wasted tokens, no more mediocre outputs.
  • →
    Built by Lee Hanna, AI Consultant at AI4U LLC
  • →
    Maps 14 platforms across 8 task stages and 4 risk levels
  • →
    Works for solo consultants, teams, and enterprises
  • →
    Generates platform-specific prompts with one click
STEP 02 / 07
📝
Step 1 — Define Your Task
Navigate to the Route Engine tab. Describe your task in plain English. Be specific about the output format, audience, and constraints.
  • 1
    Click the Route Engine tab in the navigation
  • 2
    Type your task goal (e.g., "Build a full-stack SaaS app with auth")
  • 3
    Describe your current asset and the failure point or constraint
  • 4
    Specify the required output format
STEP 03 / 07
🧠
Step 2 — Pick Stage & Risk
Select the task stage (Idea → Production → Automation) and the risk level (Low → Mission Critical). These two inputs dramatically change the routing recommendation.
  • 1
    Select Task Stage: Idea, Prototype, Broken Codebase, Production, etc.
  • 2
    Select Risk Level: Low / Medium / High / Mission Critical
  • 3
    Click Route This Task to run the analysis
  • 4
    Review the platform recommendation and confidence score
STEP 04 / 07
⚡
Step 3 — Generate Your Prompt
Navigate to Prompt Gen to build a platform-specific, expert-level prompt that extracts maximum performance from your chosen AI tool.
  • 1
    Select the target platform from the dropdown
  • 2
    Describe your goal, audience, and constraints
  • 3
    Click Generate Prompt
  • 4
    Copy to clipboard and paste into the selected platform
STEP 05 / 07
🧾
Step 4 — Verify Proof
After running the task, verify your output against the Proof Chain. Every result needs browser proof, build proof, and link proof before it's considered complete.
  • 1
    Go to the Proof Chain tab
  • 2
    Verify: Browser proof → Build proof → Link proof
  • 3
    Screenshot or log the evidence
  • 4
    Mark each proof item as PASS or document the failure
STEP 06 / 07
📊
Step 5 — Record in the Ledger
Log every task outcome in the Ledger. This builds your personal AI performance database and improves future routing decisions.
  • 1
    Go to the Ledger tab
  • 2
    Record: Task, Platform, Outcome (Worked / Partial / Failed)
  • 3
    Add why it worked, why it failed, and the artifact link
  • 4
    Tag the entry for future pattern analysis
STEP 07 / 07
🏆
Step 6 — Duplicate What Worked
You now have everything you need to operate the AI4U Routing OS at full power. Duplicate successful routes. Avoid failed ones. Visit AI4UTECH.COM for consulting, training, and the full AI4U ecosystem.
  • →
    Duplicate frameworks that produced verified results
  • →
    Create avoidance rules from documented failures
  • →
    Share your routing intelligence with your team
  • →
    Visit AI4Utech.com for advanced consulting →

Platform Selection Rules
Idea → Prototype
Use Manus Agent for big-idea-to-first-prototype. It handles multi-step orchestration, file creation, and deployment in a single session.
Prototype → Runnable
Use Replit Agent for interactive, runnable prototypes with live preview. Best for demos and client-facing MVPs.
Broken Codebase
Use Claude Code for repo repair and hardening. It reads entire codebases and applies surgical fixes with full context.
PR / Code Tasks
Use Codex CLI as a PR worker. Best for isolated, well-defined code tasks with clear acceptance criteria.
Production Deploy
Use Vercel for public app deployment. Zero-config, instant CDN, custom domains, and GitHub CI/CD built in.
Mission Critical
Use Claude Opus + Supabase + GitHub Actions for mission-critical work. Every step must produce a verifiable proof receipt.

Proof Protocol
The 7-Link Proof Chain
1
Browser Proof
The output loads in a real browser at a real URL. Screenshot the live page. Verify HTTP 200. Test in incognito mode. Record the URL and timestamp.
✓ Verified
2
Build Proof
The deployment pipeline completed without errors. Vercel shows "Ready" status. GitHub shows the commit SHA. Build logs are clean.
✓ Verified
3
Link Proof
All internal and external links resolve correctly. No 404s. Custom domain points to the correct deployment. CDN assets load.
✓ Verified
4
Data Persistence Proof
Data survives a page refresh. localStorage entries persist. (V2: Supabase records survive a session end and return on re-login.)
⏳ V2 — Supabase Upgrade
5
Email / Automation Proof
Automated workflows trigger correctly. Emails arrive. Webhooks fire. (V3: GitHub Actions proof receipts on every deploy.)
⏳ V3 — GitHub Actions
6
Screenshot / Log Proof
Visual evidence exists. Screenshots are timestamped. Console logs are clean. Error monitoring shows no critical alerts.
✓ Verified
7
Failure Fallback Proof
A documented fallback platform exists if the primary fails. The fallback has been tested. Escalation rules are defined and recorded.
✓ Documented
