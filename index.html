How To Use The AI4U Routing OS

A step-by-step guide to routing every task to the perfect AI platform — every time.

SLIDE 01 / 07
🎯

What Is the AI4U Routing OS?

The AI4U Routing OS is an intelligent decision framework that analyzes your task and routes it to the single best AI platform for that specific job. No more guessing, no more wasted tokens, no more mediocre outputs.

→Built by Lee Hanna, AI Consultant at AI4U LLC
→Maps 40+ AI platforms to 12 task categories
→Works for solo consultants, teams, and enterprises
SLIDE 02 / 07
📝

Step 1 — Describe Your Task

Navigate to the Router tab and type a plain-English description of what you need to accomplish. Be specific about the output format, audience, and any constraints.

1Click the Router tab in the navigation
2Type your task in the input field (e.g., "Write a 2,000-word blog post about AI trends")
3Select the task category that best fits your goal
SLIDE 03 / 07
🧠

Step 2 — Get Your Platform Match

The Routing OS analyzes your task against its decision matrix and returns the best AI platform with a confidence score and the exact reasoning behind the recommendation.

1Click Route This Task to run the analysis
2Review the recommended platform and confidence score
3Read the routing rationale to understand why
SLIDE 04 / 07
⚡

Step 3 — Generate Your Optimized Prompt

Navigate to the Prompt Gen tab to build a platform-specific, expert-level prompt that extracts maximum performance from your chosen AI tool.

1Select the target platform from the dropdown
2Describe your goal and set tone, format, length
3Click Generate Prompt and copy to clipboard
SLIDE 05 / 07
📊

Step 4 — Log Your Results

After running the task, log the outcome in the Ledger tab. This builds your personal AI performance database and improves future routing decisions.

1Go to the Ledger tab after completing your task
2Mark the outcome: Worked / Partial / Failed
3Add notes and tags for future reference
SLIDE 06 / 07
🔄

Step 5 — Study the Patterns

The Patterns tab reveals your personal AI usage intelligence — which platforms you over-rely on, which tasks you should delegate differently, and where you're leaving performance on the table.

1Review the platform frequency charts
2Identify your most common task categories
3Adjust your routing rules based on real outcomes

AI Routing Insights
💡
Context Window Matters

For tasks requiring analysis of long documents (200k+ tokens), Claude Opus or Gemini Ultra are the only viable options. ChatGPT's 128k context fills up fast on large codebases.

⚡
Real-Time = Perplexity

Any task requiring current events, live prices, recent news, or today's data must go to Perplexity. All other LLMs have knowledge cutoffs and will hallucinate recent facts.

🤖
Agents for Multi-Step

Tasks with 5+ sequential steps, file operations, web browsing, code execution, and deployments belong in Manus Agent — not a chat interface. Stop doing agent work in chat.

🎨
Image Quality Hierarchy

Midjourney v6 > DALL-E 3 > Stable Diffusion for photorealism. For brand consistency and style control, Midjourney with a tuned style reference is unbeatable.

📝
Long-Form = Claude

For articles over 2,000 words, Claude Sonnet/Opus consistently outperforms GPT on coherence, nuance, and following complex multi-part instructions without drift.

💻
Code Review = Claude

Claude Sonnet is the gold standard for code review, refactoring, and explaining complex codebases. For autocomplete in IDE, GitHub Copilot wins on speed and context.
